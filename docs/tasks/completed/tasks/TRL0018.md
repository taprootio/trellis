# TRL0018 — Work-a-task playbook (end-to-end task loop)

## Completed
Shipped `docs/playbooks/work-task.md` — the fixed end-to-end loop for taking a
Trellis task from `active/` to ready-for-review. The loop branches *before* any
file edit (the task rewrite and all work happen on the branch), so `main` is
never dirtied and the clean-tree check always holds. Two intentional pauses:
refinement sign-off and plan confirmation. Pairs with the code-review (TRL0017)
and PR-draft (TRL0016) playbooks; linked from AGENTS.md.

No work outstanding here: scaffolding the playbook into new repos is owned by
`trellis init` (TRL0003), and graduating it to an MCP prompt by TRL0006.

## Scope
- `docs/playbooks/work-task.md`: take a task id -> ground in conventions -> read
  and verify against the codebase -> propose refinements and wait for the user ->
  plan with commit checkpoints and confirm -> start clean and branch -> rewrite
  the task on the branch -> work end-to-end -> review via the code-review
  playbook -> fix findings -> hand off for the user's review.
- Tech-agnostic; only the loop is standardized.

## Notes
- Ports Taproot's work_task discipline into the portable Trellis recipe. init

## Risks
- Drift from the code-review / PR-draft playbooks; keep the three consistent.
