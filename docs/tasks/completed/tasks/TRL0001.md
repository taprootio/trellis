# TRL0001 — Define the Trellis backlog spec (v1)

## Completed
Shipped `SPEC.md` — the canonical, repo-agnostic **Trellis Backlog Spec
v1.0.0**: repository layout, identifiers, the active/completed/removed lifecycle,
the per-status front-matter schema, the full effort model (canonical Fibonacci
plus the optional custom effort-scale schema — labels, emoji, images, and label
aliasing), the configurable-vs-fixed config (`specVersion`, `idPrefix`,
`idWidth`, `milestones`, `priorities`, `effort`), the generated-artifact
contracts (README markers + `backlog.json` shape), the tooling/`--check`
contract, SemVer/`specVersion` versioning, and the protected-`main` +
required-check mandate.

AGENTS.md was trimmed to a summary that links to `SPEC.md`, and
`backlog.config.json` now declares `specVersion: "1.0"`.

Decisions: the spec is standalone and vendorable (no repo- or tech-specifics);
SemVer with a `specVersion` field plus a tooling mismatch warning; the
effort-scale design was settled in full now — label aliases accepted in
front-matter, the number shown alongside the label, SVG/emoji preferred with the
label as alt text, and multiple named scales selectable via `scale`.

Follow-ups: this resolves the open design questions in TRL0015, which becomes
pure implementation (generator object-form support + viewer rendering). The
`specVersion` mismatch warning lands with the generator core (TRL0002).

## Scope
- Mandate the gate: a protected `main` with the backlog check as a required status check is part of the standard (see TRL0014).
- Pin down the folder layout (`active/`, `completed/tasks/`, `removed/`), the
  front-matter schema, and the status lifecycle as a versioned spec doc.
- Define what is configurable per repo (ID prefix, milestones, priorities,
  effort scale) vs. fixed.
- State the generator contract: what `--check` validates and regenerates.

## Notes
- The bootstrap AGENTS.md in this repo is the working draft; this task hardens
  it into a standalone `SPEC.md` that other repos can vendor.

## Risks
- Over-fitting the spec to Taproot/this repo. Keep tech-stack rules out of it.
