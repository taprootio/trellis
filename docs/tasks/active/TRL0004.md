# TRL0004 ‚Äî Build the MCP server (operations as tools)

## Scope

Expose the backlog operations as MCP **tools**, reusing the TRL0002 core
(`src/backlog.mjs`) for all logic so there is one implementation behind both the
CLI and the server.

**Tools** (each returns the `backlog.json` shape or a slice of it):

| tool | args | returns |
| --- | --- | --- |
| `list_tasks` | `status?`, `milestone?` | the backlog header (`prefix`, `milestones`, `nextId`, `counts`) + `tasks[]`, filtered |
| `get_task` | `id` | the task entry **+ raw Markdown `body` + `file`** path |
| `next_id` | ‚Äî | `{ nextId }` |
| `create_task` | `title`, `summary`, `milestone`, `priority`, `effort` (number), `depends_on?`, `body?` | assigns the next id, writes `active/<id>.md` (scaffolds Scope/Notes/Risks if `body` omitted), then regenerate+validate; returns the created entry |
| `move_task` | `id`, `to` (`completed`\|`removed`), `reason` (required if `removed`), `note?`, `date?` (ISO; default = server today) | moves `active/<id>.md` ‚Üí `completed/tasks/` or `removed/`, sets `status` + `completed_on`/`removed_on`(+`removed_reason`), prepends a `## Completed`/`## Removed` note; regenerate+validate |
| `validate` | ‚Äî | `{ ok, errors, warnings }` ‚Äî config + item + marker validity (read-only) |
| `regenerate` | ‚Äî | `{ changed[], nextId, counts }` ‚Äî rewrites stale artifacts |

**Architecture** ‚Äî keep the testable logic dependency-free and isolate the SDK at
the entry point (mirrors `src/init.mjs` ‚Üî `scripts/trellis-init.mjs`):

- `src/mcp.mjs` ‚Äî zero-dependency, transport-agnostic operations over the core.
  Every function takes an explicit `repoRoot` and owns its domain validation
  (unknown id, "reason required when removing", refuse to mutate an already-invalid
  backlog). Unit-tested directly with `node --test`, no transport.
- `scripts/trellis-mcp.mjs` ‚Äî the MCP entry point; the SDK lives here only.
  `McpServer` + `StdioServerTransport`, one `registerTool` per op with a `zod`
  input shape (boundary validation + the `tools/list` JSON Schema). Each handler
  calls `src/mcp.mjs` and returns the result as JSON **text content +
  `structuredContent`**; a domain error becomes an `isError` tool result.
  Diagnostics go to **stderr** ‚Äî stdout is the JSON-RPC channel. npm: `trellis:mcp`.
- **Repo targeting:** a default root resolved at startup (`--repo <path>`, else
  `cwd`); each tool also accepts an optional `repoRoot` to override, so one server
  can serve multiple repos ‚Äî consistent with the core's explicit-`repoRoot` design.

## Notes

- This is the vendor-neutral core ‚Äî the open protocol is what makes the formula
  portable across Claude, Cursor, Windsurf, Codex, etc.
- **First runtime dependency.** Trellis was zero-dependency through TRL0003; this
  task adds `@modelcontextprotocol/sdk` + `zod` (with a committed
  `package-lock.json`). The trade was made deliberately for protocol robustness
  and to set up TRL0006 (prompts/resources). The vendored/core code
  (`src/backlog.mjs`, `src/mcp.mjs`) stays dependency-free; only the server entry
  point imports the SDK.
- `effort` is the **canonical number** here; label resolution (SPEC ¬ß6.2) is
  TRL0015 and not yet in the core, so a label would fail validation.
- Ships smoke tests (`test/mcp.test.mjs`, as TRL0003 did). Boundaries: MCP
  **prompts/resources** are TRL0006, the **`npx trellis` bin/packaging** is TRL0010,
  and the **full test matrix** is TRL0011.

## Risks

- **Write operations racing the generator.** Every mutating tool (`create_task`,
  `move_task`, `regenerate`) applies its change, then re-reads + validates the
  whole backlog and regenerates every artifact before returning. On any validation
  failure it **rolls back** to the pre-call state, so the repo is never left
  invalid or with stale generated files (mirrors init's no-partial-write ethos).
