# TRL0003 — Build the `trellis init` scaffolder

## Scope
- `trellis init` writes `docs/tasks/` scaffolding, `backlog.config.json`, the CI
  workflow, and an AGENTS.md backlog section into the target repo.
- Detect an existing backlog and no-op / merge rather than clobber.
- Prompt for (or flag) ID prefix and milestone set.

## Notes
- This is the "onboard a repo to the Taproot way" deliverable from the original
  ask.

## Risks
- Writing into repos with conflicting AGENTS.md content. Append a marked block.
