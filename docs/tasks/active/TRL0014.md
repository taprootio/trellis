# TRL0014 — Bake branch protection + required backlog check into the recipe

## Scope
- Spec (TRL0001) states the standard: `main` is protected, changes land via PR,
  and the backlog check (`backlog-readme --check`) is a required status check.
- `trellis init` (TRL0003) emits a setup helper — a `gh api` script and/or
  documented UI steps — to enable branch protection requiring that check.
- Document the equivalent for non-GitHub forges (GitLab MR pipelines, Bitbucket,
  Azure DevOps) so the recipe is not GitHub-only.
- Pin the CI job name so it can be referenced as a required check without drift.

## Notes
- Branch protection is a host repo setting, not committed code — that is why this
  is its own task rather than part of the file scaffolding in TRL0003.
- The trellis repo itself is protected immediately as an ops step, ahead of this
  task generalizing it.

## Risks
- Required-check context name drift (workflow/job rename) silently disables the
  gate. Pin the job name and document the exact context string.
