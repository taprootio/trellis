# TRL0007 — Separate the universal loop from per-repo conventions

## Scope
- Define the **per-repo conventions contract** the loop reads at runtime — a
  named set of seam points (`regenerate`, `check`/gate, `branch-naming`, repo
  `gates`, `attribution`) whose *definition* is universal and whose *values* live
  in each repo's `AGENTS.md`. Document it as `docs/playbooks/conventions.md` and
  expose it as a `trellis://` resource.
- De-leak the work-task / code-review / pr-draft playbooks so each universal step
  names a seam point (the instruction) and shows Trellis's own value only as a
  tagged example — never as the command a foreign repo should run. Fix the
  author-specific `je/` branch leak to an `<initials>/<id>/<slug>` placeholder.
- Make the contract travel: scaffold `conventions.md` via `trellis init`, and
  extend the onboarded `AGENTS.md` block to declare the seam values it was
  missing (notably `branch-naming`), resolving the `npm run` vs `npx trellis`
  contradiction between the copied playbooks and the AGENTS block.
- Prove the contract travels with a smoke test: onboard a throwaway repo via
  `init` and assert the copied playbooks carry no Trellis-only literal *as an
  instruction* and the AGENTS block declares the seam points.

## Notes
- The seam already existed in Taproot (skills read AGENTS.md). TRL0018 extracted
  the loop into portable playbooks and TRL0006 graduated them to MCP prompts that
  read the repo's files live — so "identify universal vs. Taproot-specific" is
  done at the artifact level. This task formalizes the *contract* and removes the
  Trellis/author specifics that leaked into the "universal" playbook text.
- Contract definition is universal (`conventions.md`); contract values are
  per-repo (a named "Loop contract" block in `AGENTS.md`, emitted by `init`).

## Risks
- Leaky abstractions where "universal" steps assume a stack — addressed by making
  the named seam reference the instruction and any concrete command a tagged
  example. Validation against a real, non-trivial external repo is the deeper
  proof and stays in TRL0009; the in-repo smoke test here is the first guard.
