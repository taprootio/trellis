# PL0013 — Access control / SSO for the deployed viewer

## Scope
- Put the static viewer behind Cloudflare Access (Google/GitHub/Okta/SAML), or
  publish through a host that already enforces membership.
- Do not roll custom auth for a static site.

## Notes
- Real in-app SSO only becomes relevant if the viewer gains write/edit.

## Risks
- Conflating "static" with "secured". Keep the access layer in front, not in app.
