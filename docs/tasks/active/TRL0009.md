# TRL0009 — Dogfood: onboard an existing repo end-to-end

## Scope
- Onboard one existing repo with `trellis init`.
- Work a real task via the MCP prompts; capture friction and fix the spec/CLI.

## Notes
- Taproot itself migrates its `planning/ai-backlog/` backlog as a stress test of
  the importer (its three duplicate `021`s and mixed active/completed field shapes
  make it a good one).
- **Blocked** on the backlog relocation (TRL0020) and the import engine (TRL0021):
  onboarding now lands the layout in `trellis/` and migrates Taproot's existing
  backlog via the importer rather than by hand. The MCP/profile generalization of
  the importer is TRL0022.

## Risks
- Migration of existing IDs/prefixes. The prefix-preserving import path is owned by
  TRL0021 (prefix + width, dedupe of colliding ids, dependency rewrite); this task
  proves it on a real repo and feeds back any friction.
