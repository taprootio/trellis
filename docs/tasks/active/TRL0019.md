# TRL0019 — MCP server HTTP transport (port/host) + per-repo resource addressing

## Scope
- Add a Streamable HTTP server transport alongside the existing stdio transport
  (`scripts/trellis-mcp.mjs`), selectable by flag, listening on `--port`/`--host`.
- Address resources per repo (e.g. a `trellis://{repo}/…` template) so one shared
  server can serve many repos, matching the tools/prompts `repoRoot` model.
- Guard the repo selector against path traversal, reusing the id/path hardening
  already in the tools.

## Notes
- TRL0006 built prompts/resources stateless and `repoRoot`-keyed precisely so this
  is a transport swap, not a rewrite. The current stdio server already serves many
  repos via per-call `repoRoot`; this adds the network/shared-server option.

## Risks
- Multi-tenant exposure: an HTTP server that reads arbitrary `repoRoot` paths is a
  filesystem-read surface. Scope readable roots (allowlist / base dir) and keep the
  traversal guards.
