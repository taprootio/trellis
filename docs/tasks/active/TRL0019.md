# TRL0019 — MCP server HTTP transport (port/host) + per-repo resource addressing

## Scope
- Add a Streamable HTTP server transport alongside the existing stdio transport
  (`scripts/trellis-mcp.mjs`), selectable by flag, listening on `--port`/`--host`.
- Address resources per repo (e.g. a `trellis://{repo}/…` template) and let prompts
  resolve the same repo, extending the tools' `repoRoot` model to the whole MCP
  surface so one shared server can serve many repos.
- Guard the repo selector against path traversal, reusing the id/path hardening
  already in the tools.

## Notes
- TRL0006 built the prompt/resource *builders* stateless and `repoRoot`-keyed
  precisely so this is a transport swap, not a rewrite — but bound them to the
  server's repo on the MCP surface (a prompt points at resources, which can't yet
  carry a per-call repo). This task adds the per-repo addressing the override needs
  to be safe, alongside the network/shared-server transport. The tools already
  accept `repoRoot` (their results are self-contained, so no resource-pointer
  mismatch).

## Risks
- Multi-tenant exposure: an HTTP server that reads arbitrary `repoRoot` paths is a
  filesystem-read surface. Scope readable roots (allowlist / base dir) and keep the
  traversal guards.
