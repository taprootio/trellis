# TRL0016 — PR title/description standard, template, and draft shortcut

## Scope
- The standard's `<ID>: <imperative summary>` title convention (`<ID>` = the
  repo's configured id prefix + width, per `backlog.config.json`; the colon is
  the separator — unambiguous next to an em-dash or hyphen) and the description
  sections, as documented in `.github/pull_request_template.md` and the
  `docs/playbooks/pr-draft.md` draft shortcut.
- Enforce the title standard in CI: a PR-title lint on `pull_request` that reads
  the id prefix/width from `backlog.config.json` (prefix-agnostic per TRL0007),
  so it travels into any onboarded repo unchanged.

## Notes
- The template (`.github/pull_request_template.md`) shipped with TRL0001 and the
  draft shortcut plus its `pr-draft` MCP-prompt graduation with TRL0006, so the
  only remaining deliverable here is the CI enforcement. This task also adopts
  the colon separator across the template and playbook.
- Lint shape: a zero-dependency Node script (`src/pr-title.mjs` logic +
  `scripts/pr-title-lint.mjs` CLI) reusing the same config loader as the rest of
  the toolkit, unit-tested with `node --test` — not a vendored marketplace
  action.

## Risks
- Title-lint false positives on legitimate multi-item PRs; the lint anchors on a
  leading primary id and lets the rest be named in the body.

## Follow-ups
- Scaffold the title-lint into onboarded repos via `trellis init`, paired with
  the TRL0010 package that exposes `npx trellis …` (the same way the backlog
  check graduated) — out of scope here to avoid leaning on the unshipped CLI.
