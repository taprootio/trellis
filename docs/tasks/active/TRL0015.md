# TRL0015 — Optional custom effort scales (labels, emoji, images)

## Scope
- Keep the canonical effort value as the Fibonacci number in front-matter and
  backlog.json — sorting, velocity, and cross-repo rollups depend on it.
- Add an optional effort-scale config: a 1:1 map from each value to a `label`,
  optional `emoji`, and optional `image`. Backward compatible with the current
  `effort: [1,2,3,5,8,13,21]` array form.
- Validator accepts a label as an input alias for its number (so an author may
  type `effort: Goldfish`); backlog.json always carries the resolved number plus
  label/emoji/image.
- README shows the label (plus the number for legibility); the local viewer
  (TRL0005 / TRL0008) renders emoji or image as the playful sizing indicator.

## Notes
- Map onto Fibonacci, do not replace it — the non-linear gaps (8 -> 13 -> 21 =
  "getting risky, split it") are the point; a skin keeps that math while making
  sizing approachable.
- Emoji is the cheap win: it renders in the GitHub README with no asset
  pipeline; images are a viewer upgrade.
- Generalizes beyond fish (t-shirt sizes, coffee cups, planets); fish is just a
  fun default.

## Open design decisions (resolve in refinement)
- Author input: store number only with label display-only, vs. accept label
  aliases in front-matter (lean: accept aliases).
- Show the number alongside the label in the README, or label only (lean: both).
- Image location/format and accessibility (lean: SVG/emoji, label as alt text,
  images optional).
- One active scale vs. multiple named scales selectable per repo.

## Risks
- Hiding the number harms velocity discussion — keep it available.
- Asset/theming scope creep — keep it a thin presentation layer; defer image
  rendering polish to the viewer tasks.
