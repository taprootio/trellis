# TRL0015 — Optional custom effort scales (labels, emoji, images)

## Scope

Implement the effort-scale feature already specified in [`SPEC.md`](../../../SPEC.md)
§6 (it shipped as canonical spec text in TRL0001). This task is implement-to-spec;
SPEC §6.1–6.3 is the contract, including the resolved decisions noted below.

- **Config (§6.1).** Teach the generator core to parse the effort-scale object
  form — `{ values, scales, scale }` — alongside the existing array shorthand
  (`effort: [1,2,3,5,8,13,21]` ≡ `{ values, scale: "fibonacci" }`). Validate:
  every value is mapped in the active scale, labels are unique within a scale,
  the active `scale` exists, and each entry's `label` is a non-empty string with
  optional `emoji`/`image` strings. Absent or `"fibonacci"` `scale` is the
  identity scale (label = the number).
- **Resolution (§6.2).** Front-matter `effort` MAY be the canonical number **or**
  a case-insensitive `label` from the active scale. The generator resolves a
  label to its number for active items; an unresolvable value is an actionable
  error. The canonical number stays the stored contract in front-matter and
  `backlog.json`.
- **Rendering (§6.3).** The generated README shows `label · N` (with the emoji
  inline when present) under a non-identity scale, and just `N` otherwise — the
  number stays visible for velocity/rollup math. `backlog.json` always carries
  the resolved `effort` number plus `effortLabel`, and `effortEmoji`/`effortImage`
  when present, so consumers render without the config.
- **MCP.** `create_task` accepts a label or a number for `effort`, resolving via
  the same core helper (removes the `src/mcp.mjs` guard that defers label
  resolution to this task).
- **Dogfood.** Switch this repo's `backlog.config.json` to the fish scale from
  SPEC §6.1 (covering the emoji, image-only, and label-only entry shapes) and add
  the one referenced asset, so the reference instance exercises the feature.

## Notes

- Map onto Fibonacci, do not replace it — the non-linear gaps (8 -> 13 -> 21 =
  "getting risky, split it") are the point; a skin keeps that math while making
  sizing approachable.
- Emoji is the cheap win: it renders in the GitHub README with no asset
  pipeline. Images are a viewer concern — the README renders `label · N` (+ emoji)
  but **not** images; image rendering lands with the viewer (TRL0005 / TRL0008).
- Generalizes beyond fish (t-shirt sizes, coffee cups, planets); fish is just a
  fun default.
- Resolved design decisions (were "open" in the original draft; settled by SPEC
  §6): accept label aliases in front-matter; show both `label · N`; SVG/emoji
  preferred with `label` as image alt text and images optional under
  `docs/tasks/assets/effort/`; multiple named `scales` with one active via `scale`.
- Closed items carry historical effort (§5.1, §8.3): resolve their label/emoji/
  image best-effort only when the stored number still maps under the active
  scale, otherwise emit the bare number — never re-validate or fail on it.

## Risks

- Hiding the number harms velocity discussion — keep it available (`label · N`).
- Asset/theming scope creep — keep it a thin presentation layer; defer image
  rendering polish to the viewer tasks (TRL0005 / TRL0008).
