# TRL0006 — Ship the process as MCP prompts + resources

## Scope
- Port the work-task, code-review, and pr-draft loops into MCP `prompts`,
  parameterized by repo conventions (the AGENTS.md pattern), not a baked-in stack.
- Expose the spec, conventions, config, playbooks, and the PR template as MCP
  `resources` under a `trellis://` catalog.
- Keep the equivalent Markdown playbooks for tools that read files but not prompts
  (already present under `docs/playbooks/`); the prompts are a second front-end
  over the same content, never a fork of it.

## Notes
- This answers "is MCP a good way to bring skills along" — via prompts/resources,
  yes, vendor-neutrally.
- **Serve repo files live, keyed by `repoRoot`.** Prompts and resources read the
  repo's own `SPEC.md` / `AGENTS.md` / `backlog.config.json` / `docs/playbooks/*`
  at request time, so they never drift from the playbooks and a repo's own
  customizations win. This mirrors the tools' statelessness (every op takes an
  optional `repoRoot`; `--repo` is only the default), so one server instance
  serves any number of repos — the groundwork a future shared/network server
  needs.
- Architecture follows the tool pattern: dependency-free, unit-testable builders
  in `src/prompts.mjs` (listResources / readResource / buildPrompt over the
  TRL0002 core), with the SDK wiring as thin adapters in
  `scripts/trellis-mcp.mjs`.
- The work-task prompt takes the task `id`; all prompts and resources take an
  optional `repoRoot`. Prompt bodies embed the repo's playbook text verbatim plus
  a preamble that points at the conventions resources and injects the repo's
  id-prefix vocabulary from `backlog.config.json`.

## Risks
- Prompts hardcoding one repo's tech stack. Read conventions from the repo (the
  AGENTS.md pattern) instead of baking them in. See TRL0007.
- `SPEC.md` is the Trellis project's own spec and does not travel via `init`, so
  the `spec` resource is unavailable in onboarded repos until packaging bundles a
  canonical copy (TRL0010). `readResource` reports this honestly rather than
  faking content.
- A team-shared server on a network needs an HTTP transport with `--port`/`--host`
  and per-repo resource addressing — a transport concern, not a prompts/resources
  one. Built stateless here so that lands as a transport swap; tracked separately.
