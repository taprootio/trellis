# TRL0006 — Ship the process as MCP prompts + resources

## Scope
- Port the work-task and review loops into MCP `prompts` (parameterized by repo
  conventions).
- Expose the spec and templates as MCP `resources`.
- Keep equivalent Markdown playbooks for tools that read files but not prompts.

## Notes
- This answers "is MCP a good way to bring skills along" — via prompts/resources,
  yes, vendor-neutrally.

## Risks
- Prompts hardcoding one repo's tech stack. Read conventions from the repo (the
  AGENTS.md pattern) instead of baking them in. See TRL0007.
