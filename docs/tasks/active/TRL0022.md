# TRL0022 — Expose import over MCP and ship reusable source-mapping profiles

## Scope
- Expose import over MCP as an `import` tool in `src/mcp.mjs` plus its entry point,
  sharing the TRL0021 engine (consistent with the other tools), with the same
  dry-run / report semantics and the regenerate-then-roll-back-on-failure contract
  the mutating tools already follow (TRL0004).
- Ship reusable **source-mapping profiles**: a documented schema for describing a
  source backlog, one or two built-in / example profiles, and Taproot's
  `planning/ai-backlog/` as the proven reference profile (the one TRL0009 uses).
- Wire import into onboarding: `trellis init --import <path> [--profile <name>]`
  (or a documented init→import follow-on), plus getting-started docs for
  "onboard a repo that already has a backlog."

## Notes
- The `trellis import` CLI from TRL0021 is enough to unblock the TRL0009 dogfood;
  this task generalizes the importer — named profiles plus the MCP surface — so any
  MCP-aware client can run an import and disparate repos get a describable on-ramp.
- The profiles double as regression fixtures for the TRL0011 test matrix.

## Risks
- Over-fitting the profile schema to Taproot. Validate it against at least one
  second, differently-shaped source before calling the schema stable.
