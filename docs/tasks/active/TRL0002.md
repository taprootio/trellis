# TRL0002 — Build the config-driven generator core

## Scope
- Promote `scripts/backlog-readme.mjs` into a packaged module driven entirely by
  `backlog.config.json`.
- Replace the bootstrap front-matter parser with a real YAML parser.
- Expose `validate`, `generate`, and `next-id` as library functions the CLI and
  MCP server both call.

## Notes
- The bootstrap generator already proves the contract; this is the productized
  version with tests (see TRL0011).

## Risks
- Drift between the bootstrap and productized validators. Share one schema.
