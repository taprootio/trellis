# TRL0020 — Relocate the backlog root from docs/tasks/ to trellis/

## Scope
- Change the canonical layout (SPEC §2) so the backlog root is **`trellis/`** at
  the repo root instead of `docs/tasks/`: `trellis/{active,completed/tasks,removed}/`,
  the generated `trellis/README.md`, `trellis/backlog.json`,
  `trellis/completed/index.md`, `trellis/removed/index.md`, and
  `trellis/assets/effort/`.
- Decide whether `backlog.config.json` also moves under `trellis/` (recommended —
  a single, obvious Trellis footprint) or stays at the repo root; resolve in
  refinement and update SPEC §2/§7 accordingly.
- Carry the new root through every path reference: the core (`src/backlog.mjs`),
  `trellis init` (`src/init.mjs` — skeleton paths, `MARKER_FILES`/`GENERATED`, the
  `.github/workflows/backlog.yml` body, and the AGENTS block text), the MCP server,
  resources, and prompts (`src/mcp.mjs`, `src/prompts.mjs`), the scripts, the
  copied playbooks, and all in-repo docs.
- Bump `specVersion` to **2.0** — the layout is *fixed by the spec* (§7), so moving
  the root is a breaking change per §9 — and update the generator's major-mismatch
  warning to match.
- Migrate Trellis's own backlog from `docs/tasks/` to `trellis/` with `git mv`
  (preserve history) and regenerate, keeping the repo dogfooded and `--check`-green.
- Provide a migration path for already-onboarded repos: `trellis init` (or a small
  `trellis migrate`) detects an existing `docs/tasks/` layout and relocates it to
  `trellis/` idempotently, rather than scaffolding a second backlog beside it.

## Notes
- Motivation is twofold: make a repo's adoption of Trellis obvious from its root,
  and stop colliding with static-site generators. Taproot deliberately keeps its
  backlog under `planning/` precisely because Eleventy's input dir is `docs/` (and
  it has no `.eleventyignore`), so a `docs/tasks/` backlog would be published into
  the site. A top-level `trellis/` sidesteps that for repos like it.
- Open question for refinement: make the root **configurable** (e.g. a `tasksDir`
  key defaulting to `trellis/`) versus a newly-fixed `trellis/`. Configurable is
  friendlier to disparate repos but weakens the "one obvious location" guarantee
  the rename is meant to create.
- Sequence this ahead of the importer (TRL0021), so the importer writes the new
  layout, and ahead of the dogfood (TRL0009).

## Risks
- Breaking already-onboarded repos and the packaged CLI's assumptions (TRL0010).
  Gate the change behind the major `specVersion` bump, ship the migrate path, and
  keep `--check` honest about layout drift.
- Missed path references — the pinned CI job, playbook command surfaces, the AGENTS
  block, MCP resource URIs — leaving stale `docs/tasks/` strings behind. Grep
  exhaustively; the self-migration plus smoke tests should catch regressions.
