# TRL0020 — Relocate the backlog root from docs/tasks/ to trellis/

## Scope
- Make the backlog root **configurable** via a new `tasksDir` config key that
  defaults to **`trellis/`** at the repo root (replacing the fixed `docs/tasks/`):
  `<tasksDir>/{active,completed/tasks,removed}/`, the generated
  `<tasksDir>/README.md`, `<tasksDir>/backlog.json`,
  `<tasksDir>/completed/index.md`, `<tasksDir>/removed/index.md`, and
  `<tasksDir>/assets/effort/`. Update SPEC §2/§7 to move the root from "fixed by
  the spec" to configurable-with-a-default.
- Fix the config home at **`trellis/backlog.config.json`** (moved from the repo
  root). This resolves the bootstrap problem a configurable root creates — the
  tool always finds the config at a known path, then reads `tasksDir` from it to
  locate the task tree. So: config home fixed, task tree configurable.
- Carry the new root through every path reference: the core (`src/backlog.mjs` —
  `paths()` is the central seam, plus the generator's display strings), `trellis
  init` (`src/init.mjs` — skeleton paths, `MARKER_FILES`/`GENERATED`, the config
  write location, existing-config detection, the `.github/workflows/backlog.yml`
  body, and the AGENTS block text), the MCP server, resources, and prompts
  (`src/mcp.mjs`, `src/prompts.mjs`), the scripts, the copied playbooks, the tests,
  and all in-repo docs.
- Bump `specVersion` to **2.0** — moving the root and making it configurable
  changes the fixed layout (§2) and the fixed/configurable split (§7), a breaking
  change per §9 — and keep the generator's major-mismatch warning correct.
- Migrate Trellis's own backlog from `docs/tasks/` to `trellis/` with `git mv`
  (preserve history), move `backlog.config.json` to `trellis/`, set
  `specVersion 2.0`, and regenerate — keeping the repo dogfooded and `--check`-green.

## Notes
- Motivation is twofold: make a repo's adoption of Trellis obvious from its root,
  and stop colliding with static-site generators. Taproot deliberately keeps its
  backlog under `planning/` precisely because Eleventy's input dir is `docs/` (and
  it has no `.eleventyignore`), so a `docs/tasks/` backlog would be published into
  the site. A top-level `trellis/` default sidesteps that, and a configurable
  `tasksDir` lets a repo like Taproot repoint the task tree (e.g. to `planning/`)
  without moving where the config lives.
- The open question from the original draft — configurable root vs newly-fixed
  `trellis/` — is resolved **configurable**: a `tasksDir` key defaulting to
  `trellis/`. The "one obvious location" guarantee is preserved by the default
  plus the fixed `trellis/backlog.config.json` config home, while disparate repos
  keep an escape hatch.
- The generated `backlog.config.json` **omits** `tasksDir`, relying on the
  `trellis/` default; the default is documented in SPEC §7 for repos that want to
  override it.
- No migration tooling is needed: there are no already-onboarded repos yet, so
  this is a greenfield layout decision rather than a relocation of live installs.
- Sequence this ahead of the importer (TRL0021), so the importer writes the new
  layout, and ahead of the dogfood (TRL0009).

## Risks
- Missed path references — the pinned CI job, playbook command surfaces, the AGENTS
  block, MCP resource URIs, the test fixtures — leaving stale `docs/tasks/` strings
  behind. Grep exhaustively; the regenerate + `--check` plus the test suite should
  catch regressions.
- Reintroducing a bootstrap loop if any consumer derives the config path from
  `tasksDir`. The config path MUST stay fixed at `trellis/backlog.config.json`,
  independent of `tasksDir`.
