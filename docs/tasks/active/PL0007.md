# PL0007 — Separate the universal loop from per-repo conventions

## Scope
- Identify what in Taproot's work_task/reviewed_by_god is universal (loop shape,
  plan→refine→slice→review→close) vs. Taproot-specific (Codex, EF Core, proto,
  branch naming).
- Define the per-repo conventions contract the loop reads at runtime.

## Notes
- The seam already exists in Taproot (skills read AGENTS.md); this formalizes it.

## Risks
- Leaky abstractions where "universal" steps assume a stack. Validate against a
  non-Taproot repo (PL0009).
