# TRL0021 — Build the backlog import engine (disparate schemas → Trellis)

## Scope
- Add a reusable, dependency-free import core (`src/import.mjs`, mirroring
  `src/init.mjs`): read a source backlog at an arbitrary `sourceRoot` and emit
  Trellis items into a target's current layout (`trellis/`, post-TRL0020), then run
  the TRL0002 core to validate and regenerate. Explicit `targetRoot`/`sourceRoot`,
  no process-wide state, so the CLI and a future MCP tool (TRL0022) share one
  implementation — `planImport` (read-only) / `applyImport` (writes + regenerates,
  rolls back on any failure).
- Drive the conversion from a **declarative, JSON-serializable mapping** so
  disparate repos are supported (and TRL0022 can build named profiles of the same
  shape): where source items live (active / completed / removed dirs + a filename
  pattern), how to locate each field (a YAML key, a bold-inline `**Field:**`, a
  header line like `Status:` / `Area:`, the H1, or the filename) with per-field
  fallbacks, enum remaps (priority casing; source milestones → configured
  milestones), and how to derive `title` and `summary`.
- ID handling — **fresh-sequential assignment**: assign canonical target ids (the
  target's `prefix` / `idWidth`) from the target's `nextId` upward in deterministic
  source order, so an import is safe even into a non-empty target. Colliding source
  ids (Taproot has three `021`s) are deduped by construction — each source file
  gets a unique new id — and every `depends_on` is rewritten through the
  source-id → new-id map. A dependency on a collided (ambiguous) or unknown source
  id is a hard error. Emit an id-mapping report.
- Synthesize a one-line `summary` when the source lacks one (first sentence),
  normalize `depends_on` (`None` → `[]`), and route each item to active / completed
  / removed by its source status, parsing close metadata (a `Completed:` line, else
  `Created:`, into `completed_on`; `removed_on` / `removed_reason`) into
  front-matter.
- `--dry-run` (the CLI **default**) reports the full plan — counts, the id map, and
  per-field warnings — without writing; an explicit `--apply` writes. A real run
  leaves the source tree untouched (copy-out, never delete) and produces a
  `--check`-green backlog. Lift the canonical item serializer
  (`serializeFrontMatter` / `composeFile`) into the TRL0002 core so imported files
  match hand-authored ones. Ship `node --test` smoke coverage that imports a
  fixture and validates clean through the core.

## Notes
- Tolerate intra-repo format drift via the mapping's per-field extractors +
  fallbacks: a source whose active items use bold
  `**Priority/Effort/Milestone/Depends on:**` while ~35 older completed items use a
  `Status:` / `Area:` / `Created:` header style must import from one mapping. Effort
  already matches Trellis's Fibonacci scale; priority needs case normalization;
  milestones usually need a remap (a source that mixes maturity gates with feature
  areas — `Pre-Launch / Legal Research / MCP Servers / Post-Launch / Back-Burner` —
  which §7.1 disallows, so the mapping makes that an explicit, human-supplied
  decision; an unmapped value is a hard error, never a silent guess).
- This task ships the **generic** engine plus a `trellis import` CLI that reads a
  declarative mapping from `--mapping <file.json>`; no source-specific mapping ships
  in the code. The fixture and its mapping live under `test/`. The MCP `import` tool
  and reusable, named source profiles (including Taproot's `planning/ai-backlog/`)
  ship in TRL0022; the real Taproot import is the TRL0009 dogfood.

## Risks
- Lossy or wrong field guesses corrupting the backlog. Default to dry-run, require
  an explicit mapping for anything non-obvious, fail loudly on unresolved enums or
  ids instead of guessing, and never mutate the source.
- The dependency rewrite after dedupe is the sharp edge — a missed or ambiguous
  reference points at the wrong task. Build the full id map first, rewrite from it,
  treat a dep on a collided or unknown source id as a hard error, and validate
  `depends_on` referential integrity before declaring success.
