# TRL0021 — Build the backlog import engine (disparate schemas → Trellis)

## Scope
- Add a reusable, dependency-free import core (mirroring `src/init.mjs`): read a
  source backlog at an arbitrary path and emit Trellis items into the current
  layout (`trellis/`, post-TRL0020), then run the TRL0002 core to validate and
  regenerate. Explicit `targetRoot`/`sourceRoot`, no process-wide state, so the
  CLI and a future MCP tool (TRL0022) share one implementation.
- Drive the conversion from a **declarative mapping** so disparate repos are
  supported, not just one source shape: where source items live (active /
  completed / removed globs), how to locate each field (a YAML key, a bold-inline
  `**Field:**`, or a header line like `Status:` / `Area:`), enum remaps (priority
  casing; source milestones → configured milestones), and how to derive `title`
  and `summary`.
- ID handling — the **prefix-preserving import path** called out in TRL0009's risk:
  accept a target `prefix` / `idWidth`, map bare numeric or slug source ids to
  canonical ids, and **dedupe collisions** (Taproot has three `021`s), rewriting
  every `depends_on` reference to the reassigned ids. Emit an id-mapping report.
- Synthesize a one-line `summary` when the source lacks one (first sentence, or a
  curated index description), normalize `depends_on` (`None` → `[]`), and route
  each item to active / completed / removed by its source status, parsing close
  metadata (a `Completed:` / `Created:` line) into front-matter (`completed_on`).
- `--dry-run` reports the full plan — counts, the id map, and per-field warnings —
  without writing; a real run leaves the source tree untouched (copy-out, never
  delete) and produces a `--check`-green backlog. Ship `node --test` smoke coverage
  that imports a fixture and validates clean through the core.

## Notes
- Tolerate intra-repo format drift: Taproot's active items use bold
  `**Priority/Effort/Milestone/Depends on:**`, while ~35 older completed items use
  a `Status:` / `Area:` / `Created:` header style — the mapping must handle both.
  Effort already matches Trellis's Fibonacci scale; priority needs case
  normalization; milestones usually need a remap (Taproot mixes maturity gates with
  feature areas — `Pre-Launch / Legal Research / MCP Servers / Post-Launch /
  Back-Burner` — which §7.1 disallows, so the mapping makes that an explicit,
  human-supplied decision rather than a silent guess).
- This task ships the engine plus a `trellis import` CLI and an inline mapping good
  enough to import Taproot for the TRL0009 dogfood. The MCP `import` tool and
  reusable, named source profiles ship in TRL0022.

## Risks
- Lossy or wrong field guesses corrupting the backlog. Default to dry-run, require
  an explicit mapping for anything non-obvious, fail loudly on unresolved enums or
  ids instead of guessing, and never mutate the source.
- The dependency rewrite after dedupe is the sharp edge — a missed reference points
  at the wrong task. Build the full id map first, rewrite from it, and validate
  `depends_on` referential integrity before declaring success.
