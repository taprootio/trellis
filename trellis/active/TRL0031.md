# TRL0031 — Importer: skip generated indexes and read completed/tasks; refresh the taproot profile

## Scope
- **Source selection:** don't treat generated index artifacts (`index.md` /
  `README.md`, i.e. the files the generator owns) as importable tasks, and descend
  into a nested `completed/tasks/` layout where completed items live — not just
  `completed/*.md`. Both built-in profiles currently glob `completed/*.md`, which
  grabs the generated `completed/index.md` / `removed/index.md` (they fail
  validation for missing `milestone` / `removed_reason`) **and** misses the real
  tasks under `completed/tasks/`. These are the two dogfood "blockers."
- **Refresh the `taproot-ai-backlog` profile** to Taproot's current `docs/tasks/`
  shape: source dirs (`active`, `completed/tasks`, `removed`), current milestone
  vocabulary (`v1 Production Release`, `MCP Servers`, `Post-Launch`, `Back-Burner`)
  via `remap.milestone`, and a `defaults.removed_reason`. The dogfood's working
  hand-written mapping (taproot branch `codex/dogfood-trellis-onboarding`,
  `trellis/import-taproot-docs-tasks.mapping.json`) is the blueprint.
- Ship as patch **0.1.2** through the OIDC release pipeline (bump `package.json`
  version → merge → CI publishes).

## Notes
- Surfaced by the TRL0010 taproot dogfood: the built-in profile refused to import
  (stale milestones + index files taken as tasks), so onboarding only worked after a
  custom mapping. This task promotes that working end-state into the shipped profile
  and the engine's source handling.
- Both fixes matter beyond Taproot: any Trellis-shaped source has generated indexes,
  and nested `completed/tasks/` is the Trellis layout itself.

## Risks
- Over-exclusion: scope the index/README skip to the generator's own artifact names,
  not any `index.md` a foreign source might legitimately use as a task.
- Keep existing profiles/mappings that rely on flat `completed/*.md` working (or
  migrate them) — don't silently change what a mapping selects.
