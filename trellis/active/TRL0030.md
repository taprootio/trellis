# TRL0030 — Harden packaged onboarding after Taproot dogfood

## Scope

- Treat this as a hardening slice of TRL0010's package/onboarding work, not as
  follow-on work that waits for TRL0010 and TRL0009 to close.
- Rename the publish target from the occupied unscoped npm name `trellis` to
  `ai-trellis` and update package metadata, docs, scaffolded commands, CI
  templates, AGENTS blocks, playbooks, and TRL0010 references accordingly.
  Verification on 2026-06-29 returned npm 404 for `ai-trellis`, so it appears
  available.
- Make the packaged generate/check command target-aware. A packaged CLI must
  treat the repo being operated on as `process.cwd()` by default, with an
  explicit `--target <dir>` / `--repo <dir>` override where appropriate; it must
  not derive the target repo from `import.meta.url`, because that points at the
  installed package location once run from npm.
- Keep package assets and target repo paths separate: package-relative resources
  such as built-in profiles, scaffold templates, and playbooks still resolve
  from the installed package, while backlog reads/writes resolve from the target
  repo.
- Arrange CLI output deterministically so warnings, generated-file summaries,
  and final success lines appear in human-readable order. Avoid mixing stdout
  and stderr in a way that can print `Done` before trailing warnings; reserve
  stderr for fatal errors or deliberately group advisory warnings before the
  final success message.
- Clarify the adoption flow around source retirement: `import` remains copy-out
  and never deletes the legacy backlog. The old tree is removed only by the
  explicit `init --retire-source <path>` step after the import is reviewed and
  committed, and that command stages a `git rm` for review rather than committing.
- Add regression coverage for a packaged-style invocation from outside the
  Trellis source checkout: generate/check should operate on a fixture target
  repo, not on Trellis's own repo.

## Notes

- Taproot dogfood surfaced the package-name collision by checking the live npm
  registry: `trellis` already belongs to another project, while `ai-trellis`
  returned 404 at the time of writing.
- This does not change the core import guarantee. Leaving
  `planning/ai-backlog/` in place after import was expected; retirement is a
  separate opt-in step because it is destructive in the target worktree even
  though git preserves history.
- The target-awareness fix is not something npm packaging solves by itself. The
  current internal `scripts/backlog-readme.mjs` computes its repo root from the
  script path, which is convenient inside this repo but wrong for an installed
  CLI. The packaged command needs an explicit target-root resolution layer.

## Risks

- Renaming the package late may leave stale `npx trellis ...` examples in docs,
  generated scaffold text, tests, or completed-task history. Source docs and
  templates should be updated; historical completed task prose may be left alone
  when it is clearly describing the old state.
- `npx ai-trellis ...` may still be unavailable until the package is actually
  published, so scaffolded CI should be enabled only once the package exists and
  the dogfood check proves it calls the intended binary.
