# TRL0027 — trellis init: retire and reconcile a legacy backlog on adopt

## Scope
- **Report, don't clobber:** detect stale backlog guidance (a section pointing at
  the old backlog path or the old field names, e.g. an `AI_GUIDELINES.md`
  "AI Backlog" block) and emit it as a reconciliation checklist. `init` keeps its
  guarantee never to delete author-written prose — it only appends its own marked
  block.
- **Opt-in source retirement:** add a flag (e.g. `--retire-source`) that
  history-preservingly removes the imported source tree (`git rm`), run only as a
  late, deliberate step after the import is green and committed — never automatic
  mid-import.
- Keep both behaviours explicit and discoverable in the tool's output.

## Notes
- "init reports, agent edits": `init` flags the stale section; the onboarding agent
  (following the TRL0009 recipe) does the surgical rewrite to point at Trellis. This
  keeps a fragile heuristic from ever deleting hand-written guidance.
- Retirement is deliberately separated from import so the source tree is intact if
  anything rolls back.

## Risks
- Heuristic detection of "stale guidance" will have false positives; report-only
  (no auto-delete) keeps that safe and leaves the judgment call with the agent.
