# TRL0024 — Drop the README "Next task ID"; make backlog.json the canonical source

## Scope
- Generator (`src/backlog.mjs`, `generateArtifacts`): drop the `## Next task ID`
  block from `readmeBody`; the milestone tables stay between the markers. Leave
  `nextId` in `buildBacklogJson` untouched — `backlog.json` stays canonical.
- Spec: update §3 (ids are assigned from the `nextId` published in `backlog.json`,
  not the README) and §8.1 (the README no longer publishes the next id). Bump
  `specVersion` minor.
- Tests (`test/backlog.test.mjs`): update the README-generation assertions that
  expect the "Next task ID" section; confirm `backlog.json.nextId` is still asserted.
- Regenerate this repo (`npm run backlog:readme`) so `trellis/README.md` drops the
  section, and confirm `npm run backlog:check` is green.

## Notes — import
- The importer already sources the next id from the core
  (`nextId(data.ids, cfg)` in `src/import.mjs`), not by parsing the README — so
  removing the README section needs no import change. Verify this stays true and add
  a guarding test if one is missing.
- `trellis init`'s README skeleton only emits the generated markers (no literal
  "Next task ID"), so no skeleton change is needed — confirm.

## Risks
- Browsing the repo on a forge loses the at-a-glance next id in the README.
  Mitigated: `backlog.json` carries it and the MCP `next_id` tool returns it — the
  audience already uses both.
- This edits normative spec text (§3/§8.1); keep the wording precise that
  `backlog.json` is now the published source of the next id.
