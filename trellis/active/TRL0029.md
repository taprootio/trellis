# TRL0029 — Importer: strip the source id prefix from imported titles

## Scope
- In the importer's title resolution, when the H1 begins with a token matching the
  item's source id (e.g. `# 001 README Truth Pass` for source id `001`), drop that
  leading token from the stored `title`.
- Strip only when the token matches the source id / the filename's leading numeric
  prefix, so a legitimately number-leading title is left intact.
- Drive it from the mapping (a title `strip`/`stripIdPrefix` option), default on for
  the numeric-prefix profiles (`taproot-ai-backlog`).

## Notes
- Dogfood friction (TRL0009): every imported Taproot title kept its old number
  ("001 README Truth Pass", "047 Design the Espalier MCP Server") because the `h1`
  extractor takes the heading verbatim. Cosmetic, but it shows on all imported items.

## Risks
- Be conservative — strip only an id-shaped leading token that matches the source id,
  never a meaningful leading word or number, to avoid mangling real titles.
