# TRL0034 — Init: drop the placeholder example member from the scaffolded team.json

## Scope
- `init` scaffolds `trellis/team.json` with an active `example` member. Replace it
  with an empty roster plus inline guidance (or a clearly-marked placeholder that
  doesn't validate as an active member), so it's obviously a stub to fill in rather
  than looking like a real teammate until edited.

## Notes
- Cosmetic finding from the TRL0010 taproot dogfood.

## Risks
- Keep the scaffolded file valid against the team.json schema (SPEC §7.2) — an empty
  roster must still pass `check`, and any task `owner`/`collaborators` validation must
  still behave when the roster starts empty.

## Outcome
Chose the "clearly-marked placeholder" option over an empty roster: the scaffolded
`team.json` now ships one **inactive** member, "Example teammate (replace or delete)".
It still shows the shape (`handle`/`name`/`email`/`status`) but, being inactive, can't
own active items and plainly reads as a stub rather than live config — staying
`--check`-green (no task owns it). The existing stub test was updated to assert the
member is inactive. Ships in the bundled 0.1.2 patch.
