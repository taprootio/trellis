# TRL0024 ‚Äî Drop the README "Next task ID"; make backlog.json the canonical source

## Completed
Dropped the generated **Next task ID** section from the README. `generateArtifacts`
now fills the milestone markers with the milestone tables alone (`readmeBody = block`).
`buildBacklogJson`'s `nextId` is untouched, so `backlog.json` is the single canonical
source ‚Äî the generator still computes `next` for `backlog.json` and its CLI/MCP return
value, just not for the README body.

Normative spec updated and bumped **2.2 ‚Üí 2.3.0**: ¬ß3 assigns ids from the `nextId`
in the generated `backlog.json` (¬ß8.2), and ¬ß8.1 states the README no longer publishes
the next id. The wording stays **tool-agnostic** (it cites `backlog.json` only, not
the reference MCP `next_id` tool). `SPEC_VERSION`, this repo's `backlog.config.json`,
and the AGENTS prose were bumped to match; regenerating dropped the section from
`trellis/README.md` with `backlog:check` green.

Tests: added guards that the generated README carries no "Next task ID" and that
`buildBacklogJson(...).nextId` is still emitted; added an import guard that a second
import into a non-empty target allocates ids from the core `nextId` (not the README);
updated the version-tied `init` assertion and the `history` fixtures to 2.3.

Verified (no change needed): the importer is already core-sourced (`nextId(data.ids,
cfg)`, `src/import.mjs`) and `trellis init`'s README skeleton emits markers only. No
follow-ups.

## Scope
- Generator (`src/backlog.mjs`, `generateArtifacts`): the `readmeBody` becomes just
  the milestone `block` ‚Äî drop the `## Next task ID` section; the milestone tables
  stay between the markers. Keep `const next` and the `nextId: next` return value
  (callers and `buildBacklogJson` still use it). Leave `nextId` in
  `buildBacklogJson` untouched ‚Äî `backlog.json` stays canonical.
- Spec: update ¬ß3 (ids are assigned from the `nextId` published in the generated
  `backlog.json` (¬ß8.2), not the README) and ¬ß8.1 (the README no longer publishes
  the next id; `backlog.json` carries it). Keep the normative wording
  **tool-agnostic** ‚Äî cite `backlog.json` only, not the reference MCP `next_id`
  tool (SPEC.md is vendored into any repo; the MCP server is the reference impl's
  concern, so that note lives in Risks below).
- Version: bump the spec **minor, 2.2 ‚Üí 2.3**, in SPEC.md (`Version:` line),
  `SPEC_VERSION` (`src/backlog.mjs`), `trellis/backlog.config.json`, and the
  "targets `specVersion ‚Ä¶`" line in AGENTS.md.
- Regenerate this repo (`npm run backlog:readme`) so `trellis/README.md` drops the
  section, and confirm `npm run backlog:check` is green.

## Notes ‚Äî tests
- `test/backlog.test.mjs` has **no** existing "Next task ID" assertion to update
  (and does not assert `nextId` today), so this *adds* regression guards rather than
  editing one: the generated README no longer contains "Next task ID", and
  `buildBacklogJson(...).nextId` is still emitted.
- The version bump has two test couplings: `test/init.test.mjs` asserts
  `cfg.specVersion === "2.2"` (what `trellis init` writes) ‚Üí update to `"2.3"`; two
  `test/history.test.mjs` fixtures carry `specVersion: "2.2"` ‚Üí bump to `"2.3"` for
  honesty (harmless either way ‚Äî the major matches, so no warning).

## Notes ‚Äî import
- The importer already sources the next id from the core
  (`nextId(data.ids, cfg)` in `src/import.mjs`), not by parsing the README ‚Äî so
  removing the README section needs no import change (verified). Add a guard:
  import into a **non-empty** target and assert the assigned ids continue from the
  core `nextId` (today's import tests only cover empty, freshly-init'd targets).
- `trellis init`'s README skeleton (`readmeSkeleton`, `src/init.mjs`) emits only the
  generated markers (no literal "Next task ID"), so no skeleton change is needed
  (verified).

## Risks

  Mitigated: `backlog.json` carries it and the MCP `next_id` tool returns it ‚Äî the
  audience already uses both.
- This edits normative spec text (¬ß3/¬ß8.1); keep the wording precise that
  `backlog.json` is now the published source of the next id.
