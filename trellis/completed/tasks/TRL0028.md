# TRL0028 — trellis init: scaffold process docs into the Trellis home, not docs/

## Completed
Relocated the scaffolded process docs out of `docs/` and into the fixed Trellis
home, so `trellis init` is non-invasive — it never writes into a directory a
repo's static-site generator might publish (the TRL0009 Taproot friction, where
the onboarded playbooks leaked into the Espalier Eleventy build and needed a
`.eleventyignore` workaround).

`trellis init` now scaffolds the four playbooks to `trellis/playbooks/*` and the
branch-protection recipe to `trellis/branch-protection.md`, alongside
`backlog.config.json` / `team.json` in the fixed config home (`CONFIG_DIR`) and
independent of `tasksDir`. The destinations derive from `CONFIG_DIR`, so the docs
travel with the config, not the task tree; the generator scans only `active/`,
`completed/tasks/`, and `removed/`, so the new `trellis/playbooks/` folder is
ignored by generation. The `.github/` files (workflow, PR template) stay where
forge convention requires. The generated AGENTS.md TRELLIS block, the workflow
comment, and the init CLI's closing message now point at the `trellis/` paths
(`COPY_FILES` / `agentsBlock` in `src/init.mjs`).

This repo dogfoods the layout it emits: the playbooks and recipe moved to
`trellis/playbooks/` and `trellis/branch-protection.md` (history-preserving
`git mv`), and every reference followed — this repo's `AGENTS.md`, the MCP
resources/prompts (`src/prompts.mjs`), the PR template, the pr-title lint, and the
`test/init.test.mjs` expectations. The playbook bodies are unchanged but for their
own path references; their relative links are depth-preserved (`docs/` and
`trellis/` are both one level deep, so `../../` and sibling links still resolve).
No spec change — the scaffold destination is init UX, not part of the backlog
format. `docs/import.md` (developer docs, not a scaffolded artifact) stays put.

## Scope
- Relocate the scaffolded process docs from `docs/` to the fixed Trellis home:
  `docs/playbooks/*` → `trellis/playbooks/*`, `docs/branch-protection.md` →
  `trellis/branch-protection.md`.
- Point the generated AGENTS.md TRELLIS block (and the CI/workflow comments) at the
  new paths.
- Leave the `.github/` files (`workflows/backlog.yml`, `pull_request_template.md`)
  where forge convention requires — those are not a docs collision.
- The Trellis home is the fixed `trellis/` (where `backlog.config.json` and
  `team.json` already live, independent of `tasksDir`), so the docs travel with the
  config, not the task tree. The generator scans only `active/`, `completed/tasks/`,
  and `removed/`, so a `trellis/playbooks/` folder is ignored by generation.

## Notes
- Dogfood friction (TRL0009): onboarding Taproot dropped the playbooks into `docs/`,
  which Eleventy publishes for the Espalier site — they leaked into the build and
  needed a `.eleventyignore` workaround. `docs/` is too commonly a published
  directory for init to write into.
- Goal: everything onboarding adds (outside the conventional `.github/` files) lives
  inside the single `trellis/` folder Trellis owns, so init stays non-invasive.

## Risks
- Update the init tests and any docs that reference `docs/playbooks/`; the playbook
  contents are unchanged — only their scaffold destination and the AGENTS references.
