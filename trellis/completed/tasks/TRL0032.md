# TRL0032 — Importer: preserve sound source ids, with a configured new-id floor

## Scope
- **Preserve (opt-in `--preserve-ids`):** keep an item's own id when it's sound — the
  source id matches the target format exactly (`<idPrefix><idWidth digits>`) and isn't
  already taken (by another preserved item or an existing target id). Default (no flag)
  stays today's fresh-sequential allocation — backward-compatible.
- **Gap + floor:** preserved ids keep their number; anything reassigned (a collision,
  or an id that doesn't match the format) is allocated fresh from `max(existing)+1`
  upward — into the **gap**. Import allocation **bypasses** the floor. A configured
  **`nextIdFloor`** (new `backlog.config.json` key) is recorded so that *organic*
  `nextId` = `max(floor, max+1)` — the next hand-created task starts above the imported
  band. Floor is **auto-derived** (next multiple of 1000 above the max preserved id)
  or set with **`--id-floor <N>`**.
- **Collisions:** two sources claiming one id → keep the first, reassign the rest into
  the gap, **report** each remap. Never overwrite. `depends_on` rewrites through the
  existing `bySourceId` map (preserved → itself; reassigned → new).
- **Prose refs (report-only):** when an id changes, **warn** for items whose body text
  still names a remapped id.
- **Core** (`src/backlog.mjs`): `nextId(ids, cfg)` honors `cfg.nextIdFloor`; config
  validation accepts it (optional non-negative integer, `< 10^idWidth`). **SPEC.md**:
  document the key + bump **specVersion 2.3 → 2.4** (additive); update this repo's
  `backlog.config.json`.
- **Wiring:** `--preserve-ids` / `--id-floor` through the CLI (`scripts/trellis-import.mjs`),
  the `init --import` on-ramp, and the MCP `import` tool. Import writes `nextIdFloor`
  into the target's `backlog.config.json` in preserve mode. Tests + `docs/import.md`.

## Notes
- Motivated by the TRL0010 dogfood: fresh-sequential assignment (`TR00031` → `TR00001`)
  broke the repo's identity and left body text pointing at the wrong ids.
- Maintainer's design: reserve the imported band, floor the organic `nextId` above it,
  leave the gap for collisions. Decisions: config key core-honored (spec 2.4);
  auto-derived floor with a `--id-floor` override.

## Risks
- The "sound" bar is exact-format-match + uniqueness; a mismatched prefix/width or a
  duplicate falls back to reassignment (reported), never importing a malformed id.
- `nextId` is currently `max+1`; the floor changes it to `max(floor, max+1)`. Keep it
  backward-compatible — a repo with no `nextIdFloor` is unchanged — and keep import
  allocation floor-agnostic so imported items fill the gap rather than jumping to the
  floor.
- Auto-derived floor must fit `idWidth` (`< 10^idWidth`); guard/warn on overflow.

## Outcome
Shipped opt-in `--preserve-ids`: an item keeps its own id when it matches the target
format exactly and is free; collisions (keep-first) and mismatches reassign into the
gap above the imported band, all reported. A `nextIdFloor` config key (SPEC 2.4;
`nextId` = `max(floor, max+1)`) is auto-derived (next 1000 above the band) or set via
`--id-floor`, recorded in `backlog.config.json` transactionally (rolled back on
failure) so organic ids begin above the band. Import allocation stays floor-agnostic,
so the default path is byte-identical. Stale body prose-refs to a reassigned id are
warned (report-only). Wired through the CLI, `init --import`, and the MCP tool + schema.
Fixture + tests (preserve/collision/mismatch/floor/prose-ref); docs. Independent review
clean (two doc fixes). Ships in the bundled 0.2.0 patch.
