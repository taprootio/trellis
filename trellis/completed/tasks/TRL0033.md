# TRL0033 — Importer: stop truncating synthesized summaries at a line break

## Scope
- The `summary.strategy: firstSentence` currently stops at the first newline, so a
  wrapped opening sentence is truncated — e.g. `summary: Add a real generator test
  command, make sure generator tests run in CI, and` (cut mid-sentence at a soft wrap).
- Treat soft-wrapped lines within a paragraph as one flowed string and end the summary
  at the first sentence terminator (`.` / `!` / `?`) or a sensible length cap — not at
  a `\n`.

## Notes
- Surfaced by the TRL0010 taproot dogfood (`completed/tasks/TR00029.md` had a
  truncated summary).

## Risks
- Don't flow across blank-line paragraph breaks, list items, or headings — only across
  soft wraps inside a single paragraph — or the summary will swallow unrelated text.

## Outcome
`synthSummary` (`src/import.mjs`) now flows the first prose paragraph before taking its
first sentence, so a wrapped opening sentence is captured in full; a paragraph with no
terminator still falls back to the first line (behavior unchanged), so existing
summaries are untouched. Fixture + regression test. Ships in the bundled 0.1.2 patch.
