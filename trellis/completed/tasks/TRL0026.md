# TRL0026 — Importer: resolve legacy close-dates and effort faithfully

## Completed
Gave the importer an honest fallback path for the metadata legacy backlogs omit,
without letting git leak into the gated core.

Close dates now resolve through a chain — date header → the source file's last git
commit date → an optional `defaults.completed_on`/`removed_on` floor → a hard error
— instead of failing the moment a header is absent. Git is confined to
`src/import.mjs` behind an **injected** date resolver (the default reads the *source*
repo via `execFileSync`, mirroring `src/history.mjs`); the generator and `--check`
never read git, so the gate stays reproducible (SPEC §8.4). Effort resolves from a
real `Effort:`/`Size:` signal (the profile's extractor chains `Effort → Size`)
remapped through a new `remap.effort`; with no signal it falls to `defaults.effort`,
and on a closed item — where that value is frozen history — it is flagged estimated.

Every inferred value is surfaced, never silently authored: `summary.provenance`
counts git-dated and effort-estimated items, each also carries a per-item warning,
and the CLI prints the totals. The Taproot profile chains `Effort → Size`,
`docs/import.md` documents the chain, `remap.effort`, the floor default, and the
provenance reporting, and a header-less, size-only fixture drives it end-to-end
(with a real-git integration test for the production resolver).

Follow-up: a higher-quality per-item effort estimate (an agent reading each
no-signal item) stays with the onboarding recipe (TRL0009), not the
dependency-free importer.

## Scope
- Add a **git-commit-date fallback** for `completed_on` (and `removed_on`) when a
  source item has no parseable date header. Resolution order is: date header →
  the source file's most-recent commit author-date (`git log -1 --follow
  --date=short`, read against the *source* repo) → an optional
  `defaults.completed_on` / `defaults.removed_on` floor → hard error. Never
  fabricate a close date.
- Add a **`Size:`→`effort`** path: the profile's `effort` extractor falls back to a
  `Size` label, and the engine gains **`remap.effort`** (mirroring
  `remap.priority/milestone/owner`) so a foreign size (an off-scale number, or
  `S`/`M`/`L`) resolves to a canonical effort value instead of the flat default.
- **Flag estimates explicitly:** when a close-date is git-derived or comes from a
  floor default, or effort falls to `defaults.effort` (no `Effort:`/`Size:` signal),
  emit a per-item warning and a count in the import summary
  ("N git-dated, M effort-estimated") so nothing silently passes as authored.
- **Provenance lives in the import report, not the item.** Derived values are
  surfaced in the summary + warnings (reviewed before `--apply`), not written as a
  new front-matter field — volatile git data stays out of the gated item files
  (SPEC §8.4). (Extra keys are tolerated by the serializer, but we deliberately
  don't use one here.)
- Import-time only — git is confined to `src/import.mjs` behind an **injected,
  testable** date resolver; the deterministic generator and `--check` never read git
  (SPEC §8.4).
- Update `profiles/taproot-ai-backlog.json` and `docs/import.md`; add a header-less,
  size-only fixture to the import tests.

## Notes
- Proven against Taproot: a dry run refuses because 14 completed items carry no
  `Completed:`/`Created:` header, and several inline-legacy items use `**Size:** N`
  rather than `**Effort:**`. This task is the fix that friction points to.
- Closed-item effort/priority are historical and non-gated, so an estimate is
  cosmetic-but-honest; the flag is what keeps it from being mistaken for an
  authored value.
- Git access mirrors `src/history.mjs` (TRL0025): `execFileSync("git", [...])` with
  an arg array, no shell; a non-repo / shallow / uncommitted source yields `null`
  and degrades, never throws.
- A higher-quality, per-item effort estimate (an agent reading each no-signal item)
  lives in the onboarding recipe (TRL0009), not in the dependency-free CLI.

## Risks
- Git reads at import time must degrade gracefully — missing git, a non-repo or
  shallow source, an uncommitted or single-commit history all make the resolver
  return `null`, falling through to the configured floor default (flagged
  estimated) when set. "Never fail the import" holds when git resolves the date or
  a floor default is configured; with neither, the importer refuses loudly rather
  than invent a completion date.
