# TRL0031 ‚Äî Importer: skip generated indexes; add a general trellis profile

## Scope
- **Engine** (`src/import.mjs`, `discoverSources`/`listFiles`): skip the
  generator's own artifacts (`index.md`, `README.md`) so they're never selected as
  source tasks. This is the general fix for the dogfood blocker where a `*.md` glob
  grabbed `completed/index.md` / `removed/index.md` (they then failed validation for
  missing `milestone` / `removed_reason`). Profiles can keep a plain `*.md` glob ‚Äî
  no `TR*.md`-style prefix hack needed.
- **New `profiles/trellis.json`**: a general profile that imports any *Trellis-shaped*
  backlog ‚Äî standard YAML front-matter under `active/`, **`completed/tasks/`**, and
  `removed/` (mirroring Trellis's own output layout, `STATUS_DIRS.completed =
  completed/tasks`). YAML extractors, no remap, no defaults. For adopting, splitting,
  or merging Trellis backlogs.
- **Keep `taproot-ai-backlog`** (logic unchanged) but update its `description` to mark
  it the historical bold-inline (`**Field:**`) format example / `from: inline`
  extractor reference ‚Äî it no longer describes any live repo's layout.
- Tests (`test/import.test.mjs`) + docs (`docs/import.md`).

## Notes
- Surfaced by the TRL0010 dogfood: both built-in profiles grabbed generated indexes
  as tasks and missed nested `completed/tasks/`. taproot (and `../taproot-controls`)
  are Trellis-shaped YAML because taproot seeded Trellis; the bold-inline format the
  old profile targeted isn't in active use, so a general `trellis` profile is the
  reusable artifact rather than a taproot-specific one.
- **Release is bundled:** this ships with its sibling dogfood fixes (TRL0032‚ÄìTRL0034)
  as a single patch (**0.2.0**); the `package.json` bump happens once when the batch
  is ready to PR, not in this task.

## Risks
- Over-exclusion: skip only the generator's exact artifact names (`index.md`,
  `README.md`), not any `index.md` a foreign source might legitimately use as a task.
- The `trellis` profile carries no remap ‚Äî it assumes the target's vocabulary
  (milestones / priorities / effort) matches the source's. Document that a target on a
  different vocabulary configures its milestones at `init` or supplies a `--mapping`
  override.

## Outcome
Engine (`src/import.mjs`) now skips the generator's own artifacts (`README.md`,
`completed/index.md`, `removed/index.md`) by source-relative **path** in discovery, so a
`*.md` glob never imports a generated index as a task ‚Äî while a real task merely named
`index.md` still imports.
Added `profiles/trellis.json` ‚Äî a general profile for importing any Trellis-shaped
backlog (YAML front-matter under `active/`, `completed/tasks/`, `removed/`), no
remap/defaults. Reframed `taproot-ai-backlog` as the historical bold-inline reference
(logic unchanged). New `test/fixtures/trellis-shaped/**` + a dedicated test that
imports check-green while skipping the generated artifacts (reverting the skip fails
it). Docs updated. Independent review clean. Ships in the bundled **0.2.0** patch.
