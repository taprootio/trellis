# TRL0027 — trellis init: retire and reconcile a legacy backlog on adopt

## Completed
Added two report-first adopt-time behaviours to `trellis init`, both safe by
construction — they never delete or rewrite author-written prose.

`init` now scans a bounded set of root guidance files (`AGENTS.md`,
`AI_GUIDELINES.md`, `CLAUDE.md`) for stale, pre-Trellis backlog guidance and prints it
as a `reconcile (N)` checklist for the onboarding agent to rewrite by hand. The scan
excludes Trellis's own appended block and any section already pointing at the new root,
flags a backlog-ish heading (e.g. an "AI Backlog" section), and — when `--import` was
used — flags any doc that references the imported source path. It is read-only: the
flagged files are left byte-for-byte unchanged (`scanReconcile` in `src/init.mjs`).

A new `--retire-source <path>` flag history-preservingly removes an imported source
tree: it stages a `git rm -r` (git keeps the history) and leaves the deletion for the
user to review and commit — it does not scaffold, import, or commit. It refuses unless
the path is inside the repo, exists, the repo is a git work tree, and the path is
git-tracked, and it is mutually exclusive with `--import` so retirement is always a
separate, deliberate step after the import is committed (`retireSource` in
`src/init.mjs`). `--dry-run` lists the tracked files without touching git.

No spec change (init UX is not part of the backlog format); documented in
`docs/import.md` and `init --help`, covered by tests in `test/init.test.mjs`.

Two independent, opt-out-safe adopt-time behaviours layered onto `trellis init`.
Both are tooling UX, not a backlog-format change, so the spec is untouched (no
`specVersion` bump); they are documented in `docs/import.md` and `init --help`.

## Scope

### Report, don't clobber — a reconciliation checklist
- On every `init` run, scan a **bounded** set of existing root guidance files —
  `AGENTS.md`, `AI_GUIDELINES.md`, `CLAUDE.md` — for stale backlog guidance and
  emit it as a reconciliation checklist in the tool's output (`summary.reconcile`,
  printed under a `Reconcile (N):` header). Report-only: `init` keeps its guarantee
  never to delete or rewrite author-written prose — it only appends its own marked
  block.
- Targeted heuristic (precision over recall — false positives are merely noise the
  agent skips):
  - In `AGENTS.md`, the content inside Trellis's own `<!-- BEGIN/END TRELLIS -->`
    block is excluded, so init never flags the block it just wrote.
  - Flag a heading matching `/backlog/i` (e.g. an `AI_GUIDELINES.md` "AI Backlog"
    section) in author content that does **not** already point at the new backlog
    root (`<tasksDir>/`).
  - When `--import <path>` is used, also flag any scanned file that references the
    imported (now-retired) source path — the highest-precision signal.

### Opt-in source retirement — `--retire-source <path>`
- Add `--retire-source <path>` that history-preservingly removes the imported source
  tree with `git rm -r`. It **stages** the deletion and leaves it for the user to
  review and commit (mirroring how scaffold/import leave changes for review) — it does
  not create a commit. `--dry-run` lists the tracked files and writes nothing.
- Refuse (write nothing) unless the path is inside the repo, exists, the repo is a git
  work tree, and the path is git-tracked (a history-preserving removal needs a tracked
  path).
- It is **mutually exclusive with `--import`**: retirement is a separate, later run
  after the import is green and committed — never automatic mid-import. A retire-only
  run skips scaffolding.

## Notes
- "init reports, agent edits": `init` flags the stale section; the onboarding agent
  (following the TRL0009 recipe, which depends on this) does the surgical rewrite to
  point at Trellis. This keeps a fragile heuristic from ever deleting hand-written
  guidance.
- Retirement is deliberately separated from import so the source tree is intact if
  anything rolls back, and the deletion lands as its own reviewable change.
- Scope stays on the `init` CLI plus shared core functions in `src/init.mjs`; no MCP
  surface (`init` is not an MCP tool) and no change to the scaffolded AGENTS block.

## Risks
- Heuristic detection of "stale guidance" will have false positives; report-only
  (no auto-delete, byte-identical files) keeps that safe and leaves the judgment call
  with the agent.
- `--retire-source` runs `git rm`; gating on "inside the repo, exists, tracked, and a
  git work tree" — and staging without committing — keeps it from touching anything
  unexpected or unrecoverable.
